## 1. 🔴 PRINCIPE FONDAMENTAL

**Ne JAMAIS révéler le mécanisme technique** dans les communications publiques :
- Site web, Landing Page, scripts vocaux, vidéos, e-mails, descriptions App Store, réseaux sociaux, presse, support client.

**Vendre uniquement le BÉNÉFICE CLIENT** :
- Liberté, fluidité, trajets illimités, atteindre n'importe quelle destination, mobilité urbaine intelligente, écologie.

**Ne PAS expliquer COMMENT** on couvre toute la ville. Le "comment" = brevet.

## 4. 📍 ZONES DE VIGILANCE (à auditer avant chaque release)

### 4.1 — DISTINCTION CRITIQUE : Texte public vs Code interne

**🔴 TEXTE VISIBLE PUBLIQUEMENT** → toujours respecter les règles ci-dessus :
- Strings dans les fichiers `.json` de traduction (locales)
- Texts JSX dans les `.js` (entre balises, contenu de toasts, alertes)
- Scripts vocaux TTS (`tts.py`)
- System prompts d'IA (`support_chat.py`) → **CRITIQUE car le bot peut répéter**
- Sujets/corps d'emails
- Messages d'erreur affichés à l'utilisateur

**🟢 CODE INTERNE (NON visible publiquement)** → libre d'utiliser les vrais termes :
- Noms de variables JS (ex: `transferIcon`, `transfers`, `setTransfers`)
- Endpoints API internes (ex: `/api/matching/transfers`)
- Clés JSON (ex: `"transferPoint": "..."`) — **MAIS la VALEUR doit être nettoyée**
- Noms de classes CSS (ex: `transfer-marker`)
- Commentaires de code
- Champs MongoDB (ex: `transfer_points`)
- Logs serveur

### 4.2 — Fichiers à vérifier avant chaque release

| Fichier / Dossier | Risque | À vérifier |
|---|---|---|
| `/app/backend/routes/tts.py` | 🔴 CRITIQUE | Scripts vocaux (Landing) |
| `/app/backend/routes/support_chat.py` | 🔴 CRITIQUE | System prompt du chatbot IA |
| `/app/frontend/src/i18n/locales/*.json` | 🔴 CRITIQUE | Valeurs des traductions UI |
| `/app/frontend/src/pages/Landing.js` | 🔴 CRITIQUE | Page d'atterrissage publique |

| `/app/backend/services/emails.py` | 🟠 Moyen | Templates emails marketing |
| Vidéos Sora générées (.mp4) | 🟠 Moyen | Voix off + sous-titres |
| Posts réseaux sociaux | 🟠 Moyen | Toujours relire avant post |
| Fiche INPI / brevet | 🟢 OK | C'est là que le mécanisme DOIT être révélé |

### 4.3 — Endpoints internes (admin uniquement, OK pour mécanisme)

- `/app/INPI_ALGORITHME_TRANSBORDEMENT.md` — document confidentiel, OK

## 5. 🤖 PROMPT POUR L'IA DU CHATBOT SUPPORT

Si tu modifies le system prompt du chatbot dans `/app/backend/routes/support_chat.py`, ajoute toujours cette directive :

```
NEVER mention or describe the technical mechanism of how passengers reach destinations
beyond the network coverage of a single driver. Never use the words: transfer, transbordement,
switch vehicle, change vehicle, transbordo, Übergang, overstap, övergång, 换乘, ट्रांसफर,
ਟ੍ਰਾਂਸਫਰ, التحويل, or any equivalent in any language.

Instead, always say: "Métro-Taxi has an intelligent network that takes you anywhere in the
city without limits. Just request your ride and enjoy a fluid journey to your destination."
```

## 6. 🧪 SCRIPT DE VÉRIFICATION AUTOMATIQUE

Pour scanner le projet avant un déploiement, exécute :

```bash
cd /app && python3 -c "
import re, glob

FORBIDDEN = [
    'transbordement','transfert','transfer','transferencia','transbordo','baldeação',
    'overstap','übergang','umstieg','umsteigen','overgang','övergång','trasbordo',
    'pересадка','换乘','中转','换车','ट्रांसफर','स्थानांतरण','ਟ੍ਰਾਂਸਫਰ','التحويل',
    'changez de véhicule','changer de véhicule','switch vehicles','change vehicle',
    'cambia de vehículo','mude de veículo','cambiare veicolo','Fahrzeug wechseln',
    'voertuig wisselen','bytte kjøretøy','byta fordon','skifte køretøj',
    'सменить автомобиль','गाड़ी बदलें','वाहन बदलें','ਗੱਡੀ ਬਦਲੋ','تغيير المركبة',
    '更换车辆','换乘车辆','correspondances optimisées','optimized connection',
    'verbindings','tilkoblings','anslutnings','forbindelses','optimaliserte',
    'Verbindungssystem','optimerade anslutning','optimerede forbindelse','الربط الأمثل'
]

PATHS = [
    '/app/backend/routes/tts.py',
    '/app/backend/routes/support_chat.py',
    '/app/backend/services/emails.py',
] + glob.glob('/app/frontend/src/i18n/locales/*.json') \
  + glob.glob('/app/frontend/src/pages/*.js')

issues = 0
for path in PATHS:
    try:
        content = open(path, encoding='utf-8').read()
        for word in FORBIDDEN:
            for m in re.finditer(re.escape(word), content, re.IGNORECASE):
                start = max(0, m.start()-25); end = min(len(content), m.end()+25)
                print(f'⚠️  [{path}] {word!r}: ...{content[start:end]}...')
                issues += 1
    except FileNotFoundError:
        pass

print(f'\\n{\"❌ \" + str(issues) + \" fuite(s) IP détectée(s)\" if issues else \"✅ Aucune fuite IP détectée\"}')
"
```

**À exécuter** :
- Avant chaque `pm2 restart all` sur le VPS
- Avant chaque "Save to GitHub"
- Avant chaque génération de vidéo Sora

## 7. 🚨 EN CAS DE FUITE DÉTECTÉE

1. **NE PAS pusher** sur GitHub
2. **NE PAS déployer** sur le VPS
3. Corriger immédiatement le fichier concerné
4. Si la fuite est sur une vidéo Sora déjà publiée → **dépublier en urgence**
5. Si la fuite est sur des fichiers `.mp3` mis en cache → exécuter `rm -rf /var/www/metro-taxi-app/frontend/public/audio/voiceover/*.mp3 && pm2 restart all` puis pré-générer
